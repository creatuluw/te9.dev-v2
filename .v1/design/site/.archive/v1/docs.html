CLI Quick Reference

All commands run in your terminal within your project folder.

Command Description
llmngn initInitialize plugin
llmngn add <text>Store a decision/note
llmngn listList all records
llmngn get <id>Get specific record by ID
llmngn query <text>Search context semantically
llmngn delete <id>Delete a record
llmngn cleanRemove expired records
llmngn statsShow database info
llmngn export -o fileBackup to JSON
llmngn import fileRestore from backup
llmngn purge --forceDelete all data

add

Store new context in your database.

llmngn add "Your text here"

Options:

  • -t, --type - Context type (decision, file_change, task, debt, command, architecture)
  • -m, --metadata - Extra data as JSON string
  • -s, --session - Session ID

Examples:

llmngn add "Using Redis for cache" --type decision
llmngn add "Fixed auth bug" --type file_change --metadata '{"file":"src/auth.ts"}'

list

Show all stored records.

llmngn list

Options:

  • -l, --limit - Maximum results (default: 100)
  • -t, --type - Filter by context type
  • -s, --session - Filter by session

Examples:

llmngn list --limit 20
llmngn list --type decision

query

Search your context semantically. Finds related concepts, not just exact keyword matches.

llmngn query "your search term"

Options:

  • -l, --limit - Maximum results
  • -t, --types - Filter by types (comma-separated)

Examples:

llmngn query "authentication" --limit 10
llmngn query "database" --types decision,architecture

export

Backup all your data to a JSON file.

llmngn export --output backup.json

The export includes all context records, metadata, vector embeddings, and expiration dates.

import

Restore data from a backup file.

llmngn import backup.json

Context Types

Context types categorize your stored information. Each type has a different retention period.

Type Use For Retention
decisionImportant architectural choices180 days
architectureSystem design notes365 days
debtTechnical debt tracking90 days
file_changeCode modifications90 days
taskTodos and reminders60 days
commandCommands you ran30 days

Storing Decisions

Store decisions as you make them during your workflow:

# Architectural decision
llmngn add "Microservices with API gateway pattern" --type architecture

# Technology choice
llmngn add "React frontend, Node.js backend, PostgreSQL database" --type decision

# Technical debt
llmngn add "Refactor legacy auth module in Q2" --type debt

# File change
llmngn add "Updated auth logic" --type file_change --metadata '{"file":"src/auth.ts"}'
💡 Pro Tip

Always include the --metadata flag with file paths when storing file changes. This helps track which files were modified.

Searching Context

LLMNGN uses semantic search powered by vector embeddings. This means you find related concepts, not just keyword matches.

# Search for database-related context
llmngn query "database"

# Search with type filter
llmngn query "caching" --type decision

# Search multiple types
llmngn query "auth" --types decision,file_change,architecture

Search tips:

  • Be specific: "Redis caching" finds better results than just "cache"
  • Use quotes: Search exact phrases when needed
  • Try synonyms: If "auth" doesn't work, try "authentication" or "login"
  • Filter by type: Narrow results with --type decision

Interactive Commands in OpenCode

Use /llmngn in OpenCode for natural language interaction:

/llmngn store this decision
/llmngn show me recent changes
/llmngn find anything about auth
/llmngn export my data
/llmngn what's in my database?

Common Workflows

Daily usage:

llmngn add "Completed feature X" --type task
llmngn list --limit 5

Weekly cleanup:

llmngn clean
llmngn stats

Before major changes:

llmngn export --output pre-refactor.json
# make changes
llmngn import pre-refactor.json  # if needed

Troubleshooting

Problem: Command not found

  • Run llmngn --version to verify installation
  • Reinstall: cd .opencode && bun install

Problem: No data showing

  • Run llmngn stats to verify database
  • Check .lancedb/ folder exists

Problem: Context not showing in OpenCode

  • Start a new OpenCode session
  • Verify data exists: llmngn list
  • Check plugin is enabled in .opencode/plugins/llmngn.json
