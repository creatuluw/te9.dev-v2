Getting Started

Up and Running in 5 Minutes

Install the plugin, store your first decision, and never lose context again.

What is LLMNGN?

LLMNGN is a context persistence plugin for OpenCode that remembers your coding decisions, changes, and patterns across sessions. No more starting from scratch every time you open a new session!

Step 1: Install the Plugin

Open your terminal or command prompt and navigate to your project folder:

$ npx degit hoipippeloi/llmngn.xyz/.opencode .opencode && cd .opencode && bun install && cd ..
For PowerShell users:
> npx degit hoipippeloi/llmngn.xyz/.opencode .opencode; cd .opencode; bun install; cd ..

This installs:

  • Plugin hooks (`.opencode/plugins/llmngn.ts`)
  • Configuration (`.opencode/plugins/llmngn.json`)
  • Interactive command (`.opencode/commands/llmngn.md`)

Step 2: Verify Installation

  1. Start OpenCode in your project
  2. The plugin auto-loads automatically
  3. No additional setup required!

Step 3: Store Your First Decision

$ llmngn add "This is my first decision note" --type decision

Verify it was stored:

$ llmngn list
Congratulations! You're ready to start using LLMNGN.

Step 4: Use the Interactive Command

In OpenCode, you can now use natural commands:

# Store a decision
/llmngn store that we decided to use Redis for caching

# List recent records
/llmngn show me my latest records

# Semantic search
/llmngn find anything about authentication

Exporting & Importing Data

Backup your context or share it with your team.

Export Your Data

$ llmngn export --output backup.json
✓ Exported 47 records to backup.json

Import Data

$ llmngn import backup.json
✓ Imported 47 records from backup.json

Team Collaboration

Share context with your team by exporting and sending the JSON file:

# Export team context
$ llmngn export --output team-context.json

# Team member imports
$ llmngn import team-context.json

Deleting Records

Manage your stored context by removing unneeded records.

Delete a Single Record

$ llmngn delete ctx_8f3h2k --force
✓ Deleted record ctx_8f3h2k

Clean Expired Records

Remove records that have passed their retention period:

$ llmngn clean
✓ Removed 12 expired records
Warning: This action removes all expired records. This cannot be undone.

Purge All Data

Remove all stored context (requires confirmation):

$ llmngn purge --force
✓ Purged all records
Warning: This deletes ALL your stored context. Make sure to export first if you need to keep any data.

FAQ

Frequently Asked Questions

Quick answers to common questions.

How do I install LLMNGN?
Run this in your project folder:
npx degit hoipippeloi/llmngn.xyz/.opencode .opencode && cd .opencode && bun install && cd ..
Do I need to configure anything?
No! LLMNGN works out of the box with sensible defaults. You can customize settings in .opencode/plugins/llmngn.json if needed.
Where is my data stored?
All data is stored locally in .lancedb/ folder in your project directory. No cloud upload.
Is my data secure?
Yes! Data stays on your machine. The plugin also auto-redacts API keys, passwords, tokens, and secrets.
How long is context kept?
Depends on type: Architecture (365 days), Decisions (180 days), File changes (90 days), Tasks (60 days), Commands (30 days).
Can I use LLMNGN offline?
Yes! Everything runs locally. No internet required after installation.
Can I share context with my team?
Yes! Export with llmngn export --output team-context.json and share the JSON file. Team members can import with llmngn import team-context.json.
Is LLMNGN free?
Yes! LLMNGN is completely free and open source (MIT license).
How does semantic search work?
LLMNGN uses vector embeddings to find conceptually related context, not just keyword matches. "Database" finds "PostgreSQL", "SQL", "data storage", etc.
How do I report a bug?
Create an issue at GitHub Issues or email support@llmngn.xyz
