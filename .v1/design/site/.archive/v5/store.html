Archive your decisions in the persistent layer. Each deposit survives session death, waiting to be retrieved when context is needed.

SAMPLE 001
01

The Basic Deposit

Store any decision with a single command. Your text is vectorized and stored for semantic retrieval.

llmngn add "Decided to use PostgreSQL for the database"

That's it. Your decision is now preserved in the strata.

decision (default)
SAMPLE 002
02

Classified Deposits

Categorize by type for organized retrieval. Each type has different retention periods.

llmngn add "Using microservices architecture" --type architecture llmngn add "Refactored authentication module" --type file_change llmngn add "Need to add rate limiting" --type task llmngn add "Tech debt: Optimize database queries" --type debt llmngn add "Ran migration script" --type command
architecture (365d) decision (180d) file_change (90d) task (60d) command (30d)
SAMPLE 003
03

Metadata Injection

Attach structured data for richer context. File paths, related issues, custom fields.

llmngn add "Updated auth logic" --type file_change --metadata '{"file":"src/auth.ts"}'
file_change
