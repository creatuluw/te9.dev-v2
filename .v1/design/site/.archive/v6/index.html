PERSIST[ENCE]

RETENTION: 30-365 DAYS

Architecture Decisions

365 days of decay resistance. Your structural choices—the database selections, the API patterns, the authentication strategies—these are the load-bearing walls of your codebase. They should not crumble.

TYPE: architecture

Decision Fragments

180 days. The choices made in meetings, in late-night coding sessions, in moments of clarity after debugging marathons. Why did we use Redis? Why that library? These fragments persist.

TYPE: decision

File Change Trails

90 days. What you touched. What you broke. What you fixed. The archaeological record of your filesystem modifications, preserved for those future sessions that will inevitably ask "what changed?"

TYPE: file_change

Technical Debt Ledger

90 days. The promises you made to yourself. "I'll refactor this later." "This is temporary." These debts compound interest. Now they're recorded.

TYPE: debt

Task Fragments

60 days. What remains undone. What was planned but never executed. The ghost tasks that haunt your codebase—now captured, awaiting resurrection.

TYPE: task

Command Echoes

30 days. What you ran. What flags you used. The forgotten incantations that made things work, preserved in short-term memory for immediate recall.

TYPE: command

Semantic Over Keyword

Vector embeddings dissolve the barrier between "auth" and "authentication" and "login" and "security". Search finds what you mean, not just what you typed.

Type Filtering

Narrow the recall. Focus on decisions only. Or architecture. Or debt. Combine. Intersect. The database bends to your specificity.

Temporal Decay

Older fragments fade. Fresh memories surface first. But the important decisions? They resist decay. Weighted by significance, not just timestamp.

SYNC[PROTOCOL]

INSTALLATION SEQUENCE
01

Open your terminal

Navigate to the project folder. The one that contains your code. The one that needs memory.

02

Execute the burial command

npx degit hoipippeloi/llmngn.xyz/.opencode .opencode cd .opencode && bun install && cd ..

This plants the memory system. Roots form in .opencode/. Data persists in .lancedb/

03

Verify the installation

llmngn stats

A response means memory is forming. Silence means the burial failed. Check the roots.

04

First memory insertion

llmngn add "Using PostgreSQL for primary database" --type decision

The --type flag determines decay rate. Architecture lives longest. Commands fade fastest.

FORGE[OPERATIONS]

CLI COMMAND REFERENCE
+

llmngn add

Insert a new memory fragment into the persistence layer. Decays based on --type.

llmngn add "Decided on Redis" --type decision
?

llmngn query

Semantic search across stored fragments. Finds meaning, not just matches.

llmngn query "database" --type architecture
≡

llmngn list

Enumerate all fragments. Filter by type. Limit by count. The complete inventory.

llmngn list --limit 20 --type decision
→
llmngn get

Retrieve a specific fragment by its identifier. Direct access.

llmngn get abc123-def456
−

llmngn delete

Remove a fragment permanently. Requires --force confirmation. Irreversible.

llmngn delete abc123 --force
∅

llmngn clean

Purge expired fragments. Safe operation. Only removes beyond retention date.

llmngn clean
⏣

llmngn stats

Database diagnostics. Record counts. Size metrics. Temporal range.

llmngn stats
↗

llmngn export

Extract all fragments to JSON. Backup. Archive. Transfer to another consciousness.

llmngn export --output backup.json
↘

llmngn import

Inject fragments from JSON backup. Restore. Merge. Synchronize across machines.

llmngn import backup.json
⚠

llmngn purge

Complete memory erasure. All fragments destroyed. Nuclear option. Export first.

llmngn purge --force

INTERROGATE[ANSWERS]

FREQUENTLY ASKED

Where does the data live?

In .lancedb/ within your project folder. Nowhere else. Your filesystem. Your control.

Is context injected automatically?

At session start. LLMNGN queries your stored fragments and surfaces relevant memories to your AI assistant. Zero configuration.

What if I delete the database folder?

All fragments lost. Unless you exported. Always export before dangerous operations.

Can I search by keyword?

Semantic search finds related concepts. "Database" finds "PostgreSQL", "SQL", "data layer". Vector embeddings understand meaning.

How long do records persist?

Architecture: 365 days. Decisions: 180 days. File changes: 90 days. Debt: 90 days. Tasks: 60 days. Commands: 30 days. Decay varies by importance.

Is LLMNGN free?

Yes. MIT licensed. Open source. No premium tier. No freemium. Free.

Does it work offline?

After installation, completely offline. No internet required. No cloud calls. Everything local.

How do I contribute?

GitHub repository. Issues welcome. Pull requests accepted. The memory system remembers contributors.
